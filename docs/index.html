Key Components

  • CertificateBinding CR: Declarative resource defining certificate lifecycle and destination rules
  • Controller: Watches CRs, orchestrates cert creation, validates TLS secrets, and syncs via plugins
  • Plugin System: Extensible architecture supporting multiple cloud providers and destinations
  • Metrics & Observability: Prometheus-compatible metrics and structured logging

Workflow

  1. Create a CertificateBinding CR specifying certificate source and destinations
  2. Controller detects the CR and creates cert-manager Certificate if needed
  3. cert-manager issues certificate and stores in TLS Secret
  4. Controller validates certificate integrity and expiry
  5. Configured plugins sync certificate to cloud vaults and K8s namespaces
  6. Status updates track sync results and failures

Quick Start

Get up and running in minutes

1

Install CertAuto Operator

Install using Kubernetes manifest or Helm chart:

kubectl apply -f https://github.com/sanmarg/CertAuto/releases/download/v0.1.0/certauto.yaml
2

Ensure cert-manager is Installed

CertAuto depends on cert-manager for certificate issuance:

kubectl apply -f https://github.com/cert-manager/cert-manager/releases/download/v1.19.2/cert-manager.yaml
3

Create a CertificateBinding

Define your certificate binding with source and destinations:

apiVersion: sanorg.in/v1
kind: CertificateBinding
metadata:
  name: my-cert
spec:
  caSecret:
    name: my-issuer
  commonName: example.com
  destinations:
    - name: kubernetes
      namespace: production
    - name: azure
      vaultName: my-vault
4

Monitor Certificate Status

Watch your certificate sync across platforms:

kubectl get certificatebindings -w
kubectl describe certificatebinding my-cert
